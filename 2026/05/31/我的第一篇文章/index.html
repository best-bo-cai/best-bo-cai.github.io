Hexo 维护个人博客的最佳实践

2026-05-31 更新于 2026-05-31
  • Hexo教程
  • Hexo
  • 运维

Hexo 是一个快速、简洁且高效的静态博客框架，非常适合用于搭建个人博客。结合 GitHub Pages 进行托管，可以完全免费地实现博客的部署和维护。

维护个人博客的最佳实践

  1. 版本控制（重要！）
    • Hexo 生成的静态文件在 public 目录，这个目录是不需要提交到 Git 仓库的。
    • 你应该提交的是源文件（source、themes、_config.yml 等）。
    • 推荐方法: 使用单独的分支或单独的仓库来管理源代码。
      • 方法一: 在 username.github.io 仓库中创建一个 hexo 分支用于存放源码。主分支（main/master）存放 hexo d 部署后的静态文件。
        • 在博客根目录执行：
        • 之后，当你修改了文章或配置，先提交并推送到 hexo 分支，然后运行 hexo d 部署到主分支。
      • 方法二: 创建一个私有仓库 blog-source 专门存放源码，公开仓库 username.github.io 存放部署后的静态文件。
        • 这种方法更清晰，源码和部署后的文件完全分离。
        • 步骤类似于方法一，只是推送到一个新创建的私有仓库。
  2. 撰写新文章:
    • hexo new "文章标题" 创建新文章。
    • 在 source/_posts 目录下编辑 .md 文件。
    • 建议使用 VS Code 安装 Markdown 插件进行编辑。
  3. 更换主题:
    • Hexo 官网有很多主题可选。
    • 下载主题（例如 NexT）：
    • 修改博客根目录下的 _config.yml 中的 theme 字段：
    • 主题通常都有自己的配置文件 themes/主题名/_config.yml 或 _config.yml，根据主题文档进行个性化配置。
  4. 配置文件管理:
    • Hexo 有两个主要的配置文件：
      • 站点配置文件：博客根目录下的 _config.yml。
      • 主题配置文件：themes/主题名/_config.yml。
    • 为了避免主题更新后配置文件被覆盖，推荐使用主题提供的“替代配置文件”功能（如果支持），或者将你的配置文件备份到你的版本控制系统中。
  5. 备份博客:
    • 如果你按照第 1 点做了版本控制，你的博客源码已经安全地备份在 GitHub 上了。
    • 如果你的电脑坏了，只需要在新电脑上克隆你的源码仓库，安装 Node.js、Git、Hexo CLI，然后运行 npm install 安装依赖，hexo g && hexo d 即可恢复博客。
  6. 图片管理:
    • Hexo 默认将图片放在 source/images 文件夹中。
    • 为了方便管理，推荐使用图床（如 PicGo、Imgur、七牛云等），然后在 Markdown 文中引用图片链接。
    • 或者使用插件 hexo-asset-image，在配置文件中设置 post_asset_folder: true，这样每篇文章都会生成一个同名文件夹，图片放在该文件夹内，可以用相对路径引用。
  7. 常用命令汇总:
    • 当遇到奇怪的问题时，尝试运行 hexo clean 后重新 hexo g && hexo d。
  8. 进阶操作:
    • 绑定自定义域名: 在 source 目录下创建一个 CNAME 文件（无后缀），写入你的域名（如 example.com）。然后在域名解析处添加 CNAME 记录指向 你的用户名.github.io。
    • 使用 CI/CD: 可以利用 GitHub Actions 实现自动部署，当你推送源码到仓库时，自动运行 hexo g && hexo d。这需要编写一个 workflow 文件，适合进阶用户。
