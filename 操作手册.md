# Hexo 双分支博客操作手册

## 一、双分支工作流概览

| 分支 | 用途 | 包含内容 |
|------|------|----------|
| `hexo-scode` | 源码分支 | `_config.yml`、`package.json`、`source/`、`themes/`、`scaffolds/`、`.gitignore` 等 |
| `main` | 部署分支 | 仅包含 `hexo g` 生成的静态文件（`public/` 目录的内容） |

> **核心原则**：日常开发在 `hexo-scode` 分支进行，`main` 分支由 `hexo deploy` 自动管理，**不要手动编辑 main 分支**。

---

## 二、日常操作

### 1. 写新文章

```bash
# 确保在源码分支
git checkout hexo-scode

# 创建新文章
hexo new "文章标题"

# 编辑 source/_posts/文章标题.md

# 本地预览
hexo clean && hexo g && hexo s

# 满意后提交源码
git add .
git commit -m "新增文章：文章标题"
git push origin hexo-scode

# 部署到 GitHub Pages
hexo clean && hexo deploy
```

### 2. 修改配置/主题

```bash
git checkout hexo-scode

# 修改 _config.yml 或 themes/ 下的文件

# 提交
git add .
git commit -m "更新配置"
git push origin hexo-scode

# 重新部署
hexo clean && hexo deploy
```

### 3. 同时需要出现在两个分支的文件

如 `source/about/index.md`、`source/tags/index.md` 等页面文件，它们属于**源码**，只在 `hexo-scode` 分支管理。`main` 分支会通过 `hexo deploy` 自动生成对应的静态版本。

---

## 三、.gitignore 配置

`hexo-scode` 分支的 `.gitignore` 应包含：

```gitignore
.DS_Store
Thumbs.db
db.json
*.log
node_modules/
public/
.deploy_git/
```

> **注意**：`public/` 和 `.deploy_git/` 必须忽略，前者是生成产物，后者是 hexo-deployer 的工作目录。

## 四、Front Matter 参考

每篇 Markdown 文章顶部的 `---` 之间的 YAML 元数据：

```yaml
---
title: 文章标题        # 文章标题（必需）
date: 2026-05-31       # 创建日期
tags:                  # 文章标签
  - 标签1
  - 标签2
categories:            # 文章分类（支持层级）
  - [分类1, 子分类]

description: 摘要描述  # 文章摘要
---
```

## 五、常见 Git 问题及解决方案

### 1. 拉取时提示"偏离的分支"

```
致命错误：需要指定如何调和偏离的分支。
```

**解决**：

```bash
git pull --rebase
```

建议永久配置（个人博客推荐）：

```bash
git config pull.rebase true
```

### 2. 拉取时有未暂存的修改

```
error: cannot pull with rebase: You have unstaged changes.
```

**解决**：

```bash
git stash
git pull --rebase
git stash pop
```

### 3. Rebase 冲突（modify/delete）

```
CONFLICT (modify/delete): _config.yml deleted in HEAD and modified in xxxxx
```

**解决**：
- 保留文件：`git add _config.yml`
- 删除文件：`git rm _config.yml`

然后继续 rebase：

```bash
git rebase --continue
```

如果弹出 Vim 编辑器：按 `:wq` 保存退出。

### 4. git stash 失败（needs merge）

```
error: xxx: needs merge
```

**解决**：

```bash
# 先解决冲突，选择保留哪一方的版本
git checkout --ours <文件>    # 保留本地版本
# 或
git checkout --theirs <文件>  # 保留远程版本

git add <文件>
git stash
```

### 5. 嵌入式 Git 仓库警告

```
warning: adding embedded git repository: themes/VSC4T/
```

**原因**：该目录内有自己的 `.git` 目录。

**解决**：

```bash
# 从 Git 索引中移除（不删文件）
git rm --cached -rf themes/VSC4T/
git rm --cached -rf .deploy_git/

# 提交
git add .
git commit -m "移除嵌入式 git 仓库"
```

### 6. 切换分支时被未跟踪文件阻止

```
error: The following untracked working tree files would be overwritten by checkout:
  package.json
  package-lock.json
```

**解决**：

```bash
# 如果这些文件不需要保留
rm package.json package-lock.json

# 然后切换
git checkout hexo-scode
```

### 7. 切换分支时被已修改文件阻止

```
error: Your local changes to the following files would be overwritten by checkout
```

**解决**：

```bash
# 暂存修改
git stash

# 切换分支
git checkout hexo-scode

# 如需恢复修改
git stash pop
```

## 六、环境恢复

如果 `package.json` 或 `node_modules` 丢失：

```bash
# 确保在 hexo-scode 分支
git checkout hexo-scode

# 从 Git 历史恢复 package.json
git show HEAD:package.json > package.json

# 重新安装依赖
npm install

# 验证
hexo version
```

## 七、完整工作流示例

```bash
# ===== 1. 开始工作 =====
git checkout hexo-scode
git pull --rebase

# ===== 2. 创建文章 =====
hexo new "我的第一篇文章"
# 编辑 source/_posts/我的第一篇文章.md

# ===== 3. 本地预览 =====
hexo clean && hexo g && hexo s
# 浏览器访问 http://localhost:4000

# ===== 4. 提交源码 =====
git add .
git commit -m "新增：我的第一篇文章"
git push origin hexo-scode

# ===== 5. 部署上线 =====
hexo clean && hexo deploy
# main 分支由 hexo-deployer 自动更新，无需手动操作
```

---

## 八、注意事项

1. **永远不要在 `main` 分支手动编辑文件**，它由 `hexo deploy` 管理
2. **每次部署前先 `hexo clean`**，避免残留旧文件
3. **主题如果是 git clone 来的**，需删除其 `.git` 目录，否则会出现嵌入式仓库警告
4. **`node_modules/` 不要提交到 Git**，换环境后 `npm install` 即可恢复
5. **定期 `git pull --rebase`**，避免本地与远程偏离过大
6. 如果 `hexo` 命令不识别，检查 `package.json` 是否存在、依赖是否已安装
